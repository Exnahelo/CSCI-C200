## 4. CSV Design

### `enrollments.csv` (program-generated)

**Columns:** `username,course_number`

**Column indices:** username=0, course_number=1

```csv
username,course_number
dhowe2025,CSCI-C200
mlopez_s,CSCI-C200
dhowe2025,CSCI-C211
```

**Rules:**

- A student can enroll in multiple courses (multiple rows with same username)
- A course can have multiple students (multiple rows with same course_number)
- The combination of username + course_number must be unique (no duplicate enrollments)

> **Column order note:** Our CSV uses `username,course_number` (index 0,1). The instructor's sample code uses the reverse order — be aware when reading reference materials.

### 5.1 `Admin.view_students()` — Implemented

```python
def view_students(self):
    from HoweUtil import load_students
    students = load_students()
    if not students:
        print("\n  No students on record.\n")
        return
    print("\n" + "─" * 70)
    print(f"  {'First Name':<15} {'Last Name':<15} {'Username':<15} {'Email':<25}")
    print("─" * 70)
    for row in students:
        # row: [first_name, last_name, username, password, email]
        # password (index 3) is intentionally excluded from display
        print(f"  {row[0]:<15} {row[1]:<15} {row[2]:<15} {row[4]:<25}")
    print("─" * 70 + "\n")
```

### 5.3 `Admin.view_enrollments()` — Implemented

```python
def view_enrollments(self):
    from HoweUtil import load_enrollments, load_courses
    enrollments = load_enrollments()
    courses = load_courses()
    if not enrollments:
        print("\n  No enrollments on record.\n")
        return
    print("\n" + "─" * 75)
    print(f"  {'Username':<20} {'Course Number':<20} {'Course Title':<35}")
    print("─" * 75)
    for row in enrollments:
        username = row[0]
        course_number = row[1]
        course_title = "Unknown Title"
        for course in courses:
            if course[0] == course_number:
                course_title = course[1]
                break
        print(f"  {username:<20} {course_number:<20} {course_title:<35}")
    print("─" * 75 + "\n")
```

### 5.4 `Student.is_enrolled(course_number)` — Implemented

```python
def is_enrolled(self, course_number):
    from HoweUtil import load_enrollments
    enrollments = load_enrollments()
    for row in enrollments:
        # row: [username, course_number]
        if row[0] == self.get_username() and row[1] == course_number:
            return True
    return False
```

---

### 5.5 `Student.enroll(course_number)` — Implemented

```python
def enroll(self, course_number):
    from HoweUtil import save_row_to_csv, ENROLLMENTS_CSV_PATH, ENROLLMENT_HEADERS
    if self.is_enrolled(course_number):
        return False   # already enrolled
    save_row_to_csv(ENROLLMENTS_CSV_PATH, ENROLLMENT_HEADERS,
                    [self.get_username(), course_number])
    return True
```

## 6. Algorithm Design

### 6.1 Expanded Admin Menu Algorithm (Phase 3 — 6 options)

```algorithm
After successful login (admin_obj is authenticated Admin instance):

Repeat:
    Display admin menu:
        1. Add a new student
        2. Add a new course
        3. View all students
        4. View all courses
        5. View all enrollments
        6. Exit

    Prompt for choice

    If choice = "1": admin_obj.add_student()
    Else if choice = "2": admin_obj.add_course()
    Else if choice = "3": admin_obj.view_students()
    Else if choice = "4": admin_obj.view_courses()
    Else if choice = "5": admin_obj.view_enrollments()
    Else if choice = "6": print "Goodbye." → sys.exit()
    Else: print "Invalid choice."

Until user exits
```

### 6.2 Student Login Algorithm (HoweStudent.py)

```algorithm
sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))
Import Student from HoweClasses
Import login_control, load_courses, STUDENTS_CSV_PATH, MAX_LOGIN_ATTEMPTS from HoweUtil

Display student module header
Print "Please log in to continue."

(success, student_obj) = login_control(STUDENTS_CSV_PATH, MAX_LOGIN_ATTEMPTS, Student)

If not success:
    Print "Access denied. Exiting."
    sys.exit()

Print "Welcome, {student_obj.get_username()}!"
Show student menu
```

> **Key:** `login_control()` uses `username_idx=2, password_idx=3` when `filepath == STUDENTS_CSV_PATH`. The same function works for both admin and student login.

### 6.4 `enroll_in_course(student_obj)` Algorithm (in HoweStudent.py)

```algorithm
Load courses from courses.csv using load_courses()

If no courses:
    Print "No courses are available for enrollment at this time."
    Return

Display numbered list of courses:
    1. CSCI-C200 — Introduction to Computers
    2. CSCI-C211 — Introduction to Computer Science
    ...

Repeat:
    Prompt for course number (or 'back' to return)

    If input = 'back': return to student menu

    Validate: entered course number exists in courses list
    If not valid: print error, re-prompt

    Call student_obj.enroll(course_number)
    If returns True: print "Successfully enrolled in [course_number]."
    If returns False: print "You are already enrolled in [course_number]."
    Return to student menu
```

## 8. Edge Cases & Validation

| Scenario | Handling |
| --- | --- |
| Admin views students but `students.csv` is empty or missing | Print "No students on record." |
| Admin views courses but `courses.csv` is empty or missing | Print "No courses on record." |
| Admin views enrollments but `enrollments.csv` is empty or missing | Print "No enrollments on record." |
| Admin views enrollments but a course number no longer exists in `courses.csv` | Display "Unknown Title" for that course |
| Student logs in with wrong credentials | Same 5-attempt lockout as admin (reuses `login_control`) |
| Student tries to enroll but no courses exist | Print "No courses available." and return to menu |
| Student enters an invalid course number | Print error and re-prompt |
| Student tries to enroll in a course they are already in | `student_obj.enroll()` returns False; print "Already enrolled." |
| `enrollments.csv` doesn't exist on first enrollment | `save_row_to_csv()` creates file with headers |
| Extra whitespace in any input | `.strip()` on all inputs |

## 12. Conceptual Summary

Phase 3 completes the admin module and opens the student module. The Admin class gains three view methods that read from CSV files and display formatted, meaningful output — including a joined enrollment view that pairs usernames with course titles. The Student class gains its core enrollment methods: `is_enrolled()` checks the CSV before writing, and `enroll()` delegates to the utility layer for the actual file write. The student executable reuses `login_control()` from Phase 1 by passing `Student` as the `user_class` parameter — the same function works for both modules because it determines column indices based on the filepath. The enrollment system is built around the uniqueness check that prevents duplicate records, keeping the CSV data clean for Phase 4's view feature.
