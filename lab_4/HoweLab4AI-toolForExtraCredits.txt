b.a) What are the differences between your program and the AI generated program? (3’)
Differences (Run 1: blueprint-based Codex vs my program):
	•	Error handling / exiting
	•	Mine: Uses a dedicated failure handler (e.g., _fail_and_exit()), with explicit exit codes (sys.exit(1) for invalid input, sys.exit(0) for user exit).
	•	AI (Codex): Typically prints an error message and returns from main() (ends the program without explicit exit codes).
	•	Validation structure
	•	Mine: Separate, named validators (e.g., get_valid_income, get_valid_choice, get_valid_spending) to keep main() clean and rubric-obvious.
	•	AI (Codex): Generic parsers (e.g., parse_float, parse_int) and keeps more validation logic inside main() rather than dedicated “get valid ___” functions.
	•	Routing robustness / fallback behavior
	•	Mine: Defensive handling for unexpected states (explicit fallback path / exit), so you never get a “mystery output.”
	•	AI (Codex): In some cases returns None or an unclear value on an unexpected path, which can lead to output like Evaluation: None in a weird edge case.
	•	Utility module style
	•	Mine: Explicit if/elif threshold checks inside each evaluator (more verbose but very direct).
	•	AI (Codex): Often uses a shared helper (e.g., _evaluate_level()) to reduce repeated code (more DRY, slightly more abstract).
