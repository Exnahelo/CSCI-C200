Algorithm Design (rubric-friendly, indented structure)
Problem Description
The program prompts the user to enter a monthly income value and a monthly spending amount for a selected budget category. The program validates both numeric inputs (not empty and numeric; income must be greater than 0; spending must be non-negative). The program displays a category menu stored in a constant dictionary and validates the user’s selection by checking membership in the dictionary keys. The program computes the spending as a percent of income, calls a category-specific evaluator function in a utility file to classify the percent as Low/Typical/High based on predefined thresholds, displays the result, and exits or repeats.

Utility File Blueprint: HoweUtil.py

Pre-Coding Checklist (Locked for Income-Based Version)
1) Locked Design Decisions
The following rules are fixed and must be followed during implementation.
A. Validation Rules (explicit)
Monthly income input
	•	Cannot be empty
	•	Must be numeric
	•	Must be > 0
	•	Invalid → print error + exit
Category spending input
	•	Cannot be empty
	•	Must be numeric
	•	Must be ≥ 0
	•	Policy (locked): Reject spending > income → print error + exit
Menu choice
	•	Must be an integer
	•	Must be a key in MENU_OPTIONS
	•	Invalid → print error + exit
B. Categories + Evaluation Levels
Categories
	•	Housing, Food, Transportation, Entertainment, Savings, Exit
Evaluation labels
	•	Low, Typical, High

2) Define Threshold Logic (Percent-of-Income)
Interpretation note: percent_of_income is a decimal fraction (e.g., 0.25 = 25%).
Housing thresholds
	•	Low: ≤ 0.25
	•	Typical: > 0.25 and ≤ 0.35
	•	High: > 0.35
Food thresholds
	•	Low: ≤ 0.09
	•	Typical: > 0.09 and ≤ 0.15
	•	High: > 0.15
Transportation thresholds
	•	Low: ≤ 0.08
	•	Typical: > 0.08 and ≤ 0.14
	•	High: > 0.14
Entertainment thresholds
	•	Low: ≤ 0.04
	•	Typical: > 0.04 and ≤ 0.09
	•	High: > 0.09
Savings thresholds
	•	Low: ≤ 0.10
	•	Typical: > 0.10 and ≤ 0.20
	•	High: > 0.20
Interpretation note: For Savings, High indicates a strong savings rate (positive outcome), not overspending.
Exit
	•	Exit is a valid menu choice handled in the main program (no utility evaluator).
(Design note: any percent above the Typical upper bound is classified as High.)

3) Decide Utility Function Return Type (Locked)
Return just the level
	•	"Low" | "Typical" | "High"

4) Finalize MENU_OPTIONS (Single Source of Truth)
Confirm:
	•	Keys are integers
	•	Values are labels
	•	Exit included
	•	You will:
	•	Print the menu from it
	•	Validate choice using its keys
	•	Display category name using its value
One-sentence explanation to include in design:
“The dictionary is the single source of truth for menu display, validation, and output labeling.”
